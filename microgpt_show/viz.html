📘 第①步：词元化（Tokenization）　神经网络只能处理数字，不能直接处理字符。 词元化把每个字符映射为一个整数 ID。 本模型的词汇表只有 27 个词元：26 个小写字母 + 1 个特殊的 [BOS/EOS]（序列开始/结束符，ID=0）。

点击任意词元 → 在右侧嵌入矩阵中高亮其对应行。
[BOS] 是"句子开始/结束"的特殊符号，训练时作为输入的第一个词元，也是生成结束的信号。
演示：文本 → Token ID 序列
为什么要加 [BOS]？ 模型需要知道"序列从哪里开始"。每个名字的前缀是 [BOS]，结尾也是 [BOS]（此时 BOS 充当 EOS）。 训练时，输入 [BOS, k, a, r, i, s] → 模型依次预测 [k, a, r, i, s, a, BOS]。
📘 第②步：嵌入（Embedding）——把离散ID变成"有意义的向量"
神经网络只能处理连续数字，不能直接用离散的 Token ID (0,1,2...)。 词嵌入 wte：每个 Token → nEmb 维向量。初始值随机 [-16, 16)（避免梯度消失/爆炸），训练后自动学习相似关系（如 a/b 向量相近）。 位置嵌入 wpe：每个位置 → nEmb 维向量。因为注意力本身不区分顺序（"I love you"和"you I love"对它一样），所以需要位置标记。 最终输入 = wte + wpe，让同一字符在不同位置有不同表示。← 点击下方词元可查看具体数值及解释

📘 第③步：注意力机制——"让当前字符回头看上文"
Q (Query)："我想知道什么？" K (Key)："我能提供什么？" V (Value)："具体信息"。 计算流程：① 输入 x 投影得到 Q/K/V → ② Q·Kᵀ/√d 算相似度 → ③ Softmax 归一化（权重和=1） → ④ 加权求和 V。 4个注意力头：每个头关注不同模式（相邻字符、长距离依赖、语义等），并行学习后拼接。 因果掩码：位置 i 只能看 ≤i 的位置（不能看未来），保证生成时的自回归特性。← 下方热力图深色=高权重

📘 第④步：损失与梯度——"衡量预测好坏 + 指示如何改进"
Loss（损失）：衡量模型预测与真实答案的差距。我们用交叉熵，因为这是分类问题（27个候选字符选一个）。 公式：L = -log P(正确字符)。如果模型给正确答案概率p=0.7，Loss=-log(0.7)=0.36；如果p=0.01，Loss=4.6（很差）。 梯度（Gradient）：∂L/∂θ，告诉每个参数"往哪调整才能让Loss变小"。负梯度=参数应增大，正梯度=应减小。 归一化 (1/n)∑Loss：不同文本长度不同，除以n得到"平均每字符损失"，才能公平比较。← 随机猜测Loss≈3.3，训练后降至1.8-2.0
损失公式
交叉熵损失（Cross-Entropy Loss）：
L = − log P(正确词元)
= − log softmax(logits)[正确ID]

logits 是 lm_head 线性层的输出（每个词元一个分数）。 Softmax 将分数转为概率（所有词元概率之和=1）。

模型输出概率越接近 1（对正确词元），Loss 越小（趋近 0）。 模型完全随机（每词元概率=1/27），Loss ≈ log(27) ≈ 3.30。
本次训练初始 Loss ≈ 3.24，最终降至约 1.8～2.0。
Adam 优化器
θ ← θ − lr · m̂ / (√v̂ + ε)
m̂：一阶动量（梯度的指数加权平均）
v̂：二阶动量（梯度平方的指数加权平均）
lr：学习率（线性衰减，从 0.01 → 0）

Adam 的优势：每个参数有自适应学习率，收敛更快更稳。
参数追踪器（选择训练步数后查看）
梯度：反向传播后该参数行第一组 16 个值的导数
参数值：当前权重值（Adam 更新后）
红=正，蓝=负，色深=幅度大。

💡 为什么损失曲线每次都一样？　因为 microgpt.js 使用固定随机种子 _seed=42，保证每次训练走相同路径，便于教学复现。
学习率 lr：参数更新步长 = lr * 梯度。初始 0.01→线性衰减→0。太大会震荡，太小会慢。Adam 优化器会自适应调整每个参数的实际步长（动量+二阶矩）。
有没有"模型文件"？　没有。viz.html 是一次训练的"录像"，权重只在运行时存在内存中，不会保存到磁盘。
